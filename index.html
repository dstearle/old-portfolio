Retrofitting with Responsive Web Design


  • Use of meta viewport tag to determine user's screen size
  • Media queries for breakpoints to accomadate window size
  • Images can scale up and down to fit better with content

Responsive
Non Responsive

Using responsive web design in my projects ensures that the sites I build are adaptable to any screen or window size the user wishes to use. In this example I retrofitted an existing site that did not originally use responsive web design by using several techniques.

In this scneario I added a meta viewport tag to determine the size of the user's interface and media queries to scale down or rearrange the content with several breakpoints. If you resize the window for the new project (labeled as Responsive) you can see how the content such as the navigation bar and images resize or rearrange to fit best into their containers where as the original (labeled as Non Responsive) will remain static and it's contents will be out of view when making the window smaller.
